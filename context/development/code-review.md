# Code Review

Last updated: 2026-02-11 (America/Sao_Paulo)

## Purpose

Provide a compact review checklist aligned with repository standards.

## Review Checklist

- Changes follow Clean Architecture boundaries.
- Naming/import style matches existing module conventions.
- No suppressed errors or unsafe shortcuts introduced.
- Tests/validation commands relevant to the change are executed.
- Docs/context are updated when behavior or architecture changes.

## PR Description Expectations

- What changed.
- Why it was needed.
- How it was validated.
- Any migration or compatibility impact.
