Server Initialization

Handles server startup, dependency injection setup, and graceful shutdown. Integrates with the infrastructure layer for configuration and DI container setup.

ยงArchitecture (Clean Architecture + Handle-based DI)

The server initialization follows a handle-based DI approach:

  1. Provider Handles (Infrastructure): RwLock wrappers for runtime-swappable providers
  2. Runtime Factory (Application): Creates domain services with providers from handles

Production providers are resolved via linkme registry using AppConfig, wrapped in handles, and can be switched at runtime via admin API.

MCB supports three operating modes:
