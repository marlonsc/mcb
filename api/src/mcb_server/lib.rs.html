1//! # MCP Context Browser Server
2//!
3//! MCP protocol server implementation for semantic code analysis using vector embeddings.
4//!
5//! **Documentation**: [`docs/modules/server.md`](../../../docs/modules/server.md) |
6//! **Protocol**: [`ADR-033`](../../../docs/adr/033-mcp-handler-consolidation.md),
7//! [`ADR-011`](../../../docs/adr/011-http-transport-request-response-pattern.md)
8//!
9//! For user guides and tutorials, see the [online documentation](https://marlonsc.github.io/mcb/).
10//!
11//! ## Features
12//!
13//! - **Semantic Search**: AI-powered code understanding and retrieval using vector embeddings
14//! - **Multi-Provider**: Support for `OpenAI`, Ollama, `FastEmbed`, `VoyageAI`, Gemini embedding providers
15//! - **Vector Storage**: `EdgeVec` (local), Milvus, Qdrant, Pinecone
16//! - **AST Parsing**: 14 programming languages with tree-sitter based code chunking
17//! - **Hybrid Search**: Combines BM25 lexical search with semantic similarity
18//!
19//! ## Quick Start
20//!
21//! ```rust,no_run
22//! use mcb_server::run;
23//!
24//! #[tokio::main]
25//! async fn main() -> Result<(), Box<dyn std::error::Error>> {
26//!     // Run with default config (XDG paths + environment)
27//!     // server_mode = false uses config to determine mode
28//!     run(None, false).await?;
29//!     Ok(())
30//! }
31//! ```
32//!
33//! ## Architecture
34//!
35//! This crate implements the transport and protocol layer for the MCP Context Browser.
36//! It depends on domain contracts and infrastructure services while remaining independent
37//! of specific provider implementations.
38//!
39//! ## Core Types
40//!
41//! The most important types for users:
42//!
43//! | Type | Description |
44//! | ------ | ------------- |
45//! | [`McpServer`] | Main server struct |
46//! | [`McpServerBuilder`] | Builder for server configuration |
47//!
48//! ## Feature Flags
49//!
50//! - `fastembed`: Local embeddings via `FastEmbed` (default)
51//! - `edgevec`: Local HNSW vector storage (default)
52//! - `milvus`: Milvus vector database support
53//! - `edgevec`: `EdgeVec` in-memory vector store
54//! - `redis-cache`: Redis distributed caching
55//! - `full`: All features enabled
