1//! # MCP Context Browser
2//!
3//! A Model Context Protocol server for semantic code analysis using vector embeddings.
4//!
5//! **Documentation**: [`docs/modules/api-surface.md`](../../../docs/modules/api-surface.md) |
6//! **Architecture**: [`ADR-001`](../../../docs/adr/001-modular-crates-architecture.md),
7//! [`ADR-013`](../../../docs/adr/013-clean-architecture-crate-separation.md)
8//!
9//! This crate is the public facade — it re-exports domain types and the server entry point.
10//!
11//! ## Features
12//!
13//! - **Semantic Code Search**: Find code by meaning using vector embeddings
14//! - **Multi-Language Support**: AST-based parsing for 12+ programming languages
15//! - **Multiple Vector Stores**: Support for various vector database backends
16//! - **Clean Architecture**: Domain-driven design with dependency injection
17//!
18//! ## Example
19//!
20//! ```rust
21//! use mcb::entities::CodeChunk;
22//!
23//! // Domain types are available through the mcb facade
24//! let chunk = CodeChunk {
25//!     id: "chunk-1".to_string(),
26//!     content: "fn main() {}".to_string(),
27//!     file_path: "example.rs".to_string(),
28//!     start_line: 1,
29//!     end_line: 1,
30//!     language: "rust".to_string(),
31//!     metadata: serde_json::json!({}),
32//! };
33//! assert_eq!(chunk.id, "chunk-1");
34//! ```
35//!
36//! ## Architecture
37//!
38//! The codebase follows Clean Architecture principles:
39//!
40//! - `mcb-domain` — Core business logic, port traits, entities
41//! - `mcb-application` — Application services and use cases
42//! - `mcb-infrastructure` — DI, config, routing, cross-cutting concerns
43//! - `mcb-providers` — Repository + external provider implementations
44//! - `mcb-server` — MCP protocol server, admin API endpoints
