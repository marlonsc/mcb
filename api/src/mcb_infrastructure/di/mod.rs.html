1//! Dependency Injection System — Manual Composition Root + linkme + Handle
2//!
3//! **Documentation**: [docs/modules/infrastructure.md](../../../../docs/modules/infrastructure.md#dependency-injection)
4//!
5//! Two-layer DI combining compile-time discovery with runtime flexibility:
6//!
7//! - **linkme registry**: `#[distributed_slice]` for compile-time provider discovery
8//! - **Handle pattern**: `RwLock<Arc<dyn T>>` for runtime provider switching
9//!
10//! ```text
11//! linkme (compile-time)       AppContext (manual wiring)     Handle-based
12//! ─────────────────────       ─────────────────────────      ────────────
13//! EMBEDDING_PROVIDERS    →    Resolver → init_app()     →   Handle (RwLock)
14//! (distributed slices)                                           ↓
15//!                                                          AdminService
16//!                                                         (switch via API)
17//! ```
18//!
19//! ## Key Principles
20//!
21//! - **Manual Composition Root**: `init_app()` in `bootstrap.rs` wires all services
22//! - **Trait-based DI**: All dependencies injected as `Arc<dyn Trait>`
23//! - **Compile-time Discovery**: linkme `#[distributed_slice]` registers providers
24//! - **Runtime Switching**: Providers can be changed via admin API + Handle
25//! - **Testability**: Default local providers (`FastEmbed`, `EdgeVec`) enable isolated testing
26//!
27//! ## For Consumer Crates (mcb-server tests, golden tests, etc.)
28//!
29//! - `create_test_dependencies()` — isolated repos + shared providers from `AppContext`
30//! - `create_memory_repository()` / `create_memory_repository_with_executor()` — standalone repos
31//! - `create_vcs_entity_repository()` — standalone VCS repo
32//!
33//! ## Config-Driven Initialization
34//!
35//! All providers initialize from `config/default.toml` + overrides.
36//! Use `TestConfigBuilder` for test-specific overrides.
37//! **NEVER import from `mcb_providers` directly outside this crate.**
