4//! Infrastructure Services
5//!
6//! Infrastructure service implementations for port traits defined in mcb-application.
7//!
8//! ## ARCHITECTURE RULE
9//!
10//! **CONCRETE TYPES ARE INTERNAL ONLY.**
11//!
12//! All implementations are composed in the DI bootstrap module.
13//! External code SHOULD use `init_app()` to get an `AppContext` with resolved services.
14//! NEVER import concrete types directly from here - use the trait abstractions.
15//!
16//! ## Exception: Admin Types
17//!
18//! `AtomicPerformanceMetrics` and `DefaultIndexingOperations` are exported
19//! because mcb-server needs them for `AdminState`. These implement traits from
20//! mcb-application but are infrastructure concerns, not external providers.
21//!
22//! ## Factory Functions
23//!
24//! Config-driven factories for contexts that don't need full `AppContext`:
25//!
26//! - `default_event_bus()` — in-process event bus (from config)
27//! - `create_test_browse_vector_store()` — in-memory vector store for browse tests
28//! - `create_test_vector_store_for_e2e()` — vector store with both provider + browser interfaces
29//!
30//! Use these instead of importing concrete types from `mcb-providers`.
31
32// Admin module - partially exported for mcb-server
33pub mod admin;
34
35/// Infrastructure factory helpers.
36pub mod factory;
37
38// Internal modules - implementations NOT exported
39// pub(crate) mod events;
40pub(crate) mod lifecycle;
