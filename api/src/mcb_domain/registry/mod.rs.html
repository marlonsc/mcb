1//! Provider Registry System (CA/DI backbone)
2//!
3//! This module implements the project's **Static Dependency Injection** pattern.
4//! Unlike dynamic DI containers that resolve at runtime via reflection,
5//! MCB uses `linkme` to perform **Link-Time Registration** (Static CA).
6//!
7//! ### Core DI Components
8//!
9//! 1. **Distributed Slices**: Global arrays declared with `#[linkme::distributed_slice]`
10//!    that serve as the registry "slots" during the build process.
11//! 2. **Provider Entries**: Metadata structs (e.g. [`embedding::EmbeddingProviderEntry`])
12//!    that store factory functions and canonical provider names.
13//! 3. **Service Resolution**: Submodules (e.g., [`services`]) define factory logic
14//!    to instantiate complex domain services using registered providers.
15//! 4. **Resolution Context**: The [`ServiceResolutionContext`] provides a shared pool
16//!    of infrastructure dependencies (Database, Config) to avoid circular imports.
17//!
18//! This approach ensures zero-cost discovery, compile-time safety (no missing imports),
19//! and strict separation between domain (interfaces) and implementations (providers).
20//!
21//! ## Architecture
22//!
23//! ```text
24//! ┌─────────────────────────────────────────────────────────────────┐
25//! │                    Static DI / Registration Flow                 │
26//! ├─────────────────────────────────────────────────────────────────┤
27//! │                                                                 │
28//! │  1. Provider defines:  #[linkme::distributed_slice(PROVIDERS)]  │
29//! │                        static ENTRY: ProviderEntry = ...        │
30//! │                              ↓                                  │
31//! │  2. Registry declares: #[linkme::distributed_slice]             │
32//! │                        pub static PROVIDERS: [Entry] = [..]     │
33//! │                              ↓                                  │
34//! │  3. Resolver queries:  PROVIDERS.iter()                         │
35//! │                              ↓                                  │
36//! │  4. Config selects:    "provider = ollama" → OllamaProvider     │
37//! │                                                                 │
38//! └─────────────────────────────────────────────────────────────────┘
39//! ```
40//!
41//! ## Usage
42//!
43//! ### Registering a Provider (in mcb-providers)
44//!
45//! ```no_run
46//! use mcb_domain::registry::embedding::{EmbeddingProviderEntry, EMBEDDING_PROVIDERS};
47//! use mcb_domain::ports::EmbeddingProvider;
48//! use std::sync::Arc;
49//!
50//! // Providers register via #[linkme::distributed_slice(EMBEDDING_PROVIDERS)]
51//! // See mcb-providers for implementation examples
52//! ```
53//!
54//! ### Resolving a Provider (in mcb-infrastructure)
55//!
56//! ```no_run
57//! use mcb_domain::registry::embedding::{EmbeddingProviderConfig, resolve_embedding_provider};
58//!
59//! fn get_provider() -> Result<(), mcb_domain::Error> {
60//!     let config = EmbeddingProviderConfig::new("null");
61//!     let provider = resolve_embedding_provider(&config)?;
62//!     println!("Using provider: {}", provider.provider_name());
63//!     Ok(())
64//! }
65//! ```
