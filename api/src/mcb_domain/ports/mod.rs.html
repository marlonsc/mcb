1//! Domain Port Interfaces
2//!
3//! **Documentation**: [`docs/modules/domain.md#port-interfaces-domain-boundaries`](../../../../docs/modules/domain.md#port-interfaces-domain-boundaries)
4//!
5//! Defines all boundary contracts between domain and external layers.
6//! Ports are organized by their purpose and enable dependency injection
7//! with clear separation of concerns.
8//!
9//! ## Architecture
10//!
11//! Ports define the contracts that external layers must implement.
12//! This follows the Dependency Inversion Principle:
13//! - High-level modules (domain) define interfaces
14//! - Low-level modules (providers, infrastructure) implement them
15//!
16//! ## Organization
17//!
18//! - **admin** - Administrative interfaces for system management and monitoring
19//! - **infrastructure/** - Infrastructure services (sync, snapshots, auth, events)
20//! - **providers/** - External service provider ports (embeddings, vector stores, search)
21//! - **services** - Application service ports (validation, etc.)
22
